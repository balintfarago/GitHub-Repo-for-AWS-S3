How to write, read, update and delete items in a DynamoDB table (Python, Node.js)

  • Balint Farago
  • 17 April, 2023

In this tutorial I'm going to show you most of the write, read, update and delete operations you can do with a DynamoDB table using Node.js and Python.

1. Create a simple DynamoDB table

Let's first create a table in DynamoDB. Open the DynamoDB console and click Create table.

Name your table Invoices. I'm going to use PK as the Partition key and SK as the Sort key to enable multiple invoices issued to the same department. Select Customize settings for Table settings and select DynamoDB Standard table. For the Read/Write capacity settings select On-demand. Click Create table.

2. Add a new item to DynamoDB

Let's add another item to the table. To do this, just create a Lambda function and provide it the permission to access your table.


import json
import boto3

def lambda_handler(event, context):
    dynamodb = boto3.resource('dynamodb')
    #table name
    table = dynamodb.Table('Invoices')

    try:
        response = table.put_item(
            Item={
                'PK': 'DEPT#IT',
                'SK': 'INVOICE#19897',
                'STATUS': 'PAID',
                'VENDOR': 'Miracle Corp',
                'ISSUED': '2023-01-31 12:33:18',
                'CONTRACT': {
                    'CONTRACT ID': '753245',
                    'DISCOUNT': '2%',
                    'CONTACT PERSON': 'Mona Lisa'
                }
            }
        )
        return response
    except Exception as e:
        print("Error inserting item:", e)
        raise e

Here's the same code in Node.js:


const AWS = require('aws-sdk');
const dynamodb = new AWS.DynamoDB.DocumentClient();

exports.handler = async (event, context) => {
    const params = {
        TableName: 'Invoices',
        Item: {
            'PK': 'DEPT#IT',
            'SK': 'INVOICE#19897',
            'STATUS': 'PAID',
            'VENDOR': 'Miracle Corp',
            'ISSUED': '2023-01-31 12:33:18',
            'CONTRACT': {
                'CONTRACT ID': '753245',
                'DISCOUNT': '2%',
                'CONTACT PERSON': 'Mona Lisa'
            }
        }
    };

    try {
        const response = await dynamodb.put(params).promise();
        console.log('Item added successfully:', response);
        return response;
    } catch (err) {
        console.error('Error adding item:', err);
        throw err;
    }
};

3. Read data from DynamoDB
3.1 Get a specific record

In DynamoDB we always need to specify the partition key. Let's get all attributes for Invoice #12532 sent to the Finance department:


import json
import boto3

def lambda_handler(event, context):
    dynamodb = boto3.resource('dynamodb')
    table = dynamodb.Table('Invoices')

    response = table.get_item(
        Key={
            "PK": "DEPT#FINANCE",
            "SK": "INVOICE#12532"
        }
    )

    return response

Here's the same code in Node.js:


const AWS = require('aws-sdk');
const dynamodb = new AWS.DynamoDB();

exports.handler = async (event, context) => {
  const params = {
    Key: {
      "PK": {S: "DEPT#FINANCE"},
      "SK": {S: "INVOICE#12532"}
    },
    TableName: "Invoices"
  };

  const response = await dynamodb.getItem(params).promise();
  return response;
};

3.2 Get all unpaid invoices from the IT department

Okay, let's make something interesting. This time we're going to get a list of all the invoices that are A) sent to the IT department and B) their status is "UNPAID":


import json
import boto3
from boto3.dynamodb.conditions import Key, Attr

def lambda_handler(event, context):
    dynamodb = boto3.resource('dynamodb')
    table = dynamodb.Table('Invoices')

    response = table.query(
        KeyConditionExpression=Key('PK').eq('DEPT#IT'),
        FilterExpression=Attr('STATUS').eq('UNPAID')
    )

    return response

Here's the same code in Node.js:


const AWS = require('aws-sdk');
const dynamodb = new AWS.DynamoDB();

exports.handler = async (event, context) => {
  const params = {
    TableName: "Invoices",
    KeyConditionExpression: 'PK = :pk',
    FilterExpression: 'STATUS = :status',
    ExpressionAttributeValues: {
      ':pk': {S: 'DEPT#IT'},
      ':status': {S: 'UNPAID'}
    }
  };

  const response = await dynamodb.query(params).promise();
  return response.Items;
};

The Node.js code throws an error and says STATUS is a reserved word. This means that the name STATUS cannot be used directly in an expression. The solution is to implement ExpressionAttributeNames and perform some other modifications:


const AWS = require('aws-sdk');
const dynamodb = new AWS.DynamoDB();

exports.handler = async (event, context) => {
  const params = {
    TableName: "Invoices",
    KeyConditionExpression: "#pk = :pkValue",
    FilterExpression: "#status = :statusValue",
    ExpressionAttributeNames: {
      "#pk": "PK",
      "#status": "STATUS"
    },
    ExpressionAttributeValues: {
      ':pkValue': { S: "DEPT#IT" },
      ':statusValue': { S: "UNPAID" }
    }
  };

  const response = await dynamodb.query(params).promise();
  return response.Items;
};

Though it looks complicated at first, it's fairly easy to understand. For example for the partition key we define KeyConditionExpression as "#pk = :pkValue" using 2 variables. Then we tell the coding engine that "#pk" equals "PK" and define ':pkValue' as { S: "DEPT#IT" }. Same for STATUS. Makes sense, right? For the complete list of reserved words, see Reserved Words in the Amazon DynamoDB Developer Guide.

3.3 Get a list of all invoices from the Sales department that were issued in 2023

Now we make one step further and try to get a list based on date, which might be of interest to you since the ISSUED attribute was defined as a String. There is no "DATE" attribute type in DynamoDB and yet AWS makes this fairly easy. Here we go:


import json
import boto3
from boto3.dynamodb.conditions import Key, Attr

def lambda_handler(event, context):
    dynamodb = boto3.resource('dynamodb')
    table = dynamodb.Table('Invoices')

    response = table.query(
        KeyConditionExpression=Key('PK').eq('DEPT#SALES'),
        FilterExpression=Attr('ISSUED').begins_with('2023')
    )

    return response

Here's the same code in Node.js:


const AWS = require('aws-sdk');
const dynamodb = new AWS.DynamoDB.DocumentClient();

exports.handler = async (event, context) => {
  const params = {
    TableName: "Invoices",
    KeyConditionExpression: "#pk = :pk",
    FilterExpression: "begins_with(ISSUED, :issued)",
    ExpressionAttributeNames: {
      "#pk": "PK"
    },
    ExpressionAttributeValues: {
      ":pk": "DEPT#SALES",
      ":issued": "2023"
    }
  };

  const response = await dynamodb.query(params).promise();
  return response.Items;
};

3.4 Use variables in the condition

Let's initiate a variable in our expression. Let's go back to 3.2 where we retrieved a list of all unpaid invoices of the IT department. Now let's make this expression generic and use a variable for the department's name:


import json
import boto3
from boto3.dynamodb.conditions import Key, Attr

def lambda_handler(event, context):
    dynamodb = boto3.resource('dynamodb')
    table = dynamodb.Table('Invoices')
    dept_name = "IT"
    response = table.query(
        KeyConditionExpression=Key('PK').eq(f"DEPT#{dept_name}"),
        FilterExpression=Attr('STATUS').eq('UNPAID')
    )

    return response

Here's the same code in Node.js:


const AWS = require('aws-sdk');
const dynamodb = new AWS.DynamoDB.DocumentClient();

exports.handler = async (event, context) => {
  const dept_name = "IT";
  const params = {
    TableName: "Invoices",
    KeyConditionExpression: "#pk = :pk",
    FilterExpression: "#status = :statusValue",
    ExpressionAttributeNames: {
      "#pk": "PK",
      "#status": "STATUS"
      
    },
    ExpressionAttributeValues: {
      ":pk": `DEPT#${dept_name}`,
      ":statusValue": "UNPAID"
    }
  };

  const response = await dynamodb.query(params).promise();
  return response.Items;
};

4. Update an item

I hope it's clear now how to insert and read data from a DynamoDB table. Let's see an example of updating an item. Let's suppose that invoice #15620 has been paid, so let's update the status to "PAID":


import json
import boto3
from boto3.dynamodb.conditions import Key, Attr

def lambda_handler(event, context):
    dynamodb = boto3.resource('dynamodb')
    table = dynamodb.Table('Invoices')

    response = table.update_item(
        Key={
            "PK": "DEPT#IT",
            "SK": "INVOICE#15620"
        },
        UpdateExpression="set #status = :status",
        ExpressionAttributeNames={
            "#status": "STATUS",
        },
        ExpressionAttributeValues={
            ":status": "PAID",
        }
    )
    
    return response

Here's the same code in Node.js:


const AWS = require('aws-sdk');
const dynamodb = new AWS.DynamoDB.DocumentClient();

exports.handler = async (event, context) => {
  const params = {
    TableName: 'Invoices',
    Key: {
      PK: 'DEPT#IT',
      SK: 'INVOICE#15620'
    },
    UpdateExpression: 'set #status = :status',
    ExpressionAttributeNames: {
      '#status': 'STATUS'
    },
    ExpressionAttributeValues: {
      ':status': 'PAID'
    }
  };
 const response = await dynamodb.update(params).promise();
    return response;
  
};

If you're wondering how to update multiple records in the table, I have bad news for you. DynamoDB does not support batch update and even with a workaround you can't update more than 25 items at a time (for this you need to scan the whole table first and write a code that updates every 25 items per request, but this is an expensive operation). This is one of the things you cannot use a NoSQL db for.

Final thoughts

As you see I specified the partition key in all of our queries. DynamoDB is a NoSQL database and is designed differently from traditional relational databases. Should you want to make a request like getting a list of all unpaid invoices, you should either use Global Secondary Indexes or an SQL database like Amazon RDS or Aurora. Also, in DynamoDB it's not possible to get the average or the sum of certain items, it's not designed to perform mathematical operations. DynamoDB it's a fantastic concept for storing all kinds of data in just one table and it makes queries lightning-fast.
